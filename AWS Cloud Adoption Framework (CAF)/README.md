O **AWS Cloud Adoption Framework (AWS CAF)** é outro conceito teórico que **cai na prova Cloud Practitioner** — e muita gente confunde com os **pilares do Well-Architected Framework**, mas são coisas diferentes.

---

## 📚 **O que é o AWS Cloud Adoption Framework (CAF)?**

> O **AWS CAF** ajuda empresas a **planejar e executar a migração para a nuvem** com sucesso.

Ele foca mais na **transformação organizacional** do que na arquitetura técnica. É tipo um **guia estratégico para empresas adotarem a nuvem de forma segura e eficiente**.

---

## 🧠 **O CAF tem 6 perspectivas:**

| Perspectiva    | Foco principal                                                    | Quem se envolve?                 |
| -------------- | ----------------------------------------------------------------- | -------------------------------- |
| **Business**   | Objetivos de negócio, retorno sobre investimento                  | Executivos, Finanças             |
| **People**     | Gestão de mudança organizacional, treinamento, papéis e cultura   | RH, Gerentes de Mudança          |
| **Governance** | Estratégias de governança, gerenciamento de portfólio, compliance | PMO, Riscos, Auditoria           |
| **Platform**   | Arquitetura da nuvem, infraestrutura, automação                   | Engenheiros de Infra e DevOps    |
| **Security**   | Gerenciamento de identidade, segurança de dados e conformidade    | Equipe de Segurança e Compliance |
| **Operations** | Operações de TI, monitoramento, suporte e gestão de incidentes    | Equipe de Operações de TI        |

---

### 🤯 Diferença resumida:

| CAF                                       | Well-Architected Framework              |
| ----------------------------------------- | --------------------------------------- |
| Adotar a nuvem na **organização**         | Construir **aplicações bem projetadas** |
| Foco em **processos, pessoas e negócios** | Foco em **arquitetura técnica**         |

---

### 🎯 Exemplo de pergunta da prova:

> **Pergunta:** O que o AWS Cloud Adoption Framework ajuda as organizações a fazer?

**A.** Avaliar se os workloads seguem boas práticas de arquitetura
**B.** Entender como a nuvem impacta áreas como negócios, pessoas e segurança
**C.** Medir o desempenho de workloads na AWS
**D.** Automatizar o deployment de workloads na AWS

✅ **Resposta correta: B**
