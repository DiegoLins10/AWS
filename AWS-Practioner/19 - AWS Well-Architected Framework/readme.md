
# 📘 README – Seção 22: Well-Architected Framework e Ecossistema (AWS Cloud Practitioner)

---

## 1️⃣ Os 6 Pilares do AWS Well-Architected Framework

O AWS Well-Architected Framework é um conjunto de boas práticas para construir arquiteturas seguras, eficientes e resilientes na AWS.

Ele é baseado em **6 pilares**:

1. Excelência Operacional
2. Segurança
3. Confiabilidade
4. Eficiência de Desempenho
5. Otimização de Custos
6. Sustentabilidade

📌 Na prova: saber os nomes e o objetivo de cada pilar.

---

## 2️⃣ Pilar 1 – Excelência Operacional

Foco:
Executar e monitorar sistemas de forma eficiente.

Princípios:

* Automação
* Monitoramento
* Melhoria contínua
* Responder a eventos

Exemplo:
Usar CloudWatch para monitoramento.

---

## 3️⃣ Pilar 2 – Segurança

Foco:
Proteger dados, sistemas e ativos.

Princípios:

* Princípio do menor privilégio
* Criptografia
* Gerenciamento de identidade (IAM)
* Proteção de dados

📌 Muito cobrado na prova.

---

## 4️⃣ Pilar 3 – Confiabilidade

Foco:
Sistema deve se recuperar de falhas.

Princípios:

* Multi-AZ
* Backup
* Escalabilidade automática
* Recuperação automática

Exemplo:
RDS Multi-AZ.

---

## 5️⃣ Pilar 4 – Eficiência de Desempenho

Foco:
Usar recursos adequados conforme necessidade.

Princípios:

* Escolher tipo correto de serviço
* Escalar automaticamente
* Usar tecnologias modernas

Exemplo:
DynamoDB para alta escala.

---

## 6️⃣ Pilar 5 – Otimização de Custos

Foco:
Evitar gastos desnecessários.

Princípios:

* Pagar apenas pelo uso
* Monitorar custos
* Usar Savings Plans/Reserved Instances
* Desligar recursos ociosos

📌 Muito comum em questões de prova.

---

## 7️⃣ Pilar 6 – Sustentabilidade

Foco:
Reduzir impacto ambiental.

Princípios:

* Eficiência energética
* Uso otimizado de recursos
* Arquiteturas eficientes

É o pilar mais recente.

---

# 📘 AWS Cloud Adoption Framework (CAF)

O AWS CAF ajuda empresas na jornada de adoção da nuvem.

Ele organiza a transformação em perspectivas:

* Business
* People
* Governance
* Platform
* Security
* Operations

📌 CAF = framework de transformação organizacional para adoção de cloud.

# 🌎 AWS Ecosystem

O ecossistema AWS inclui:

* Parceiros (AWS Partners)
* Marketplace
* Comunidade
* Suporte técnico
* Treinamentos e certificações

Ajuda empresas a implementar soluções mais rapidamente.

---

## 🧠 AWS IQ

AWS IQ conecta clientes a especialistas certificados da AWS.

Uso:

* Consultoria
* Implementação
* Suporte especializado

---

## 🛠️ AWS Serviços Gerenciados

Serviços gerenciados significam que a AWS cuida da infraestrutura.

Exemplos:

* RDS
* DynamoDB
* Lambda
* S3

Benefícios:

* Menos trabalho operacional
* Mais foco no negócio
* Alta disponibilidade automática

📌 Na prova:
Gerenciado = AWS cuida da infraestrutura.
